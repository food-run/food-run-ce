# Food Run Trademark Policy

This policy explains how the Food Run name and brand assets may and may not be used.

The open source software license for Food Run Community Edition (CE) does **not** grant trademark rights. Trademark and brand permissions are governed separately by this policy.

## 1) What this policy covers

“Food Run Marks” includes:
- the name `Food Run`
- project logos, icons, badges, and brand graphics
- distinctive visual identifiers associated with official Food Run releases or services
- confusingly similar names, logos, or branding used in a way that suggests affiliation, endorsement, continuity, or official status

## 2) What the software license does not grant

Even where the code may be used, modified, or redistributed under the applicable software license, that does **not** by itself grant permission to:
- rename your product or service using the Food Run Marks
- use Food Run logos or official visual branding
- market a fork or hosted service as if it were the official Food Run project
- imply endorsement, sponsorship, certification, or partnership without written permission

## 3) Permitted nominative and factual uses

You may use the Food Run Marks without written permission for truthful, non-confusing, referential purposes such as:
- referring to the official project by name
- linking to the official repository or website
- stating that your tool is compatible with Food Run CE
- describing an unmodified copy of the official CE release
- writing tutorials, reviews, case studies, or educational material about Food Run
- saying that your project is “a fork of Food Run CE” if that statement is accurate and not misleading

These uses must not imply that your project, product, or service is the official Food Run project unless it actually is.

## 4) Uses that require written permission

You must obtain written permission from the Project Owner before you:
- use `Food Run` in a product name, company name, app name, package name, or domain name in a source-identifying way
- use Food Run logos, icons, or official visual assets in your own product or service branding
- market a hosted service, support offering, commercial edition, or derivative product using the Food Run Marks
- use the marks in advertising, sponsorships, merchandise, partner materials, or fundraising campaigns
- present your project or service in a way that suggests official endorsement, certification, partnership, or maintained continuity

## 5) Forks and modified versions

If you publish a fork, modified version, or derivative project:
- you must use a different name and branding
- you must not use Food Run logos or confusingly similar branding
- you must not call your fork `Food Run`, `Food Run Plus`, `Food Run Enterprise`, `Food Run Cloud`, `Official Food Run`, or similar names that imply official status
- you should clearly state that your project is a fork of Food Run CE and is not the official Food Run project

Examples of generally acceptable naming:
- `MealFlow (fork of Food Run CE)`
- `Pantry Planner, based on Food Run CE`

Examples of generally unacceptable naming:
- `Food Run Pro`
- `Food Run Enterprise`
- `Food Run Hosted`
- `Official Food Run Community`
- any branding that is visually or textually likely to confuse users about source

## 6) Hosted services

If you host a service based on Food Run CE or a modified version:
- you must not brand the service as `Food Run` without written permission
- you must not imply endorsement, sponsorship, certification, or partnership
- you must clearly identify your own organization as the service provider
- you must comply with the applicable software license for the code you use

## 7) Attribution and acknowledgments

Forks and derivative projects are encouraged to acknowledge the upstream project in a factual way, for example:
- `Based on Food Run CE`
- `Forked from Food Run CE`

Such acknowledgment must remain non-confusing and must not imply official endorsement.

## 8) Donation, sponsorship, and support links

Nothing in this policy requires forks or derivative projects to display donation links, sponsorship links, or support buttons for the original maintainers unless separately agreed in writing.

Forks may voluntarily acknowledge or support the upstream project, but no such fundraising display obligation is granted or imposed by this policy alone.

## 9) Enforcement

The Project Owner may object to or require correction of any use of the Food Run Marks that is:
- misleading
- confusing
- deceptive
- reputationally harmful
- inconsistent with this policy
- likely to cause users to believe a fork, hosted service, or derivative offering is official when it is not

Permission to use the marks may be denied, limited, or revoked where appropriate.

## 10) Reserved rights

All rights in the Food Run Marks are reserved except where this policy expressly permits use.

## 11) Contact

For trademark permissions or questions, use the contact method published by the Project Owner in the repository profile, website, or governance/security documents.
