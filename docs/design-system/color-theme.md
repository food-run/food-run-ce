# Color and Theme

## Purpose

Color in Food Run should support orientation, affordance, and warmth.

The interface already has a playful-comic foundation. Color should refine that personality into something production-ready, not amplify it into noise.

## Color Goals

The color system should feel:
- Warm
- Legible
- Restrained
- Believable
- Semantic
- Supportive of the workflow

## Color Roles

Use color by role, not by whim.

Recommended roles:
- App background
- Surface background
- Elevated surface
- Primary text
- Secondary text
- Muted text
- Border or separator
- Primary accent
- Secondary accent
- Success
- Warning
- Error
- Info
- Focus ring

## Neutral Strategy

Neutrals should do most of the structural work.

Use:
- Slightly tinted neutrals
- Clear text hierarchy
- Gentle but readable surface differentiation

Avoid:
- Pure harsh grays everywhere
- Muddy low-contrast surfaces
- Overly colorful backgrounds that compete with content

## Accent Strategy

Accent color is for:
- Primary-action emphasis
- Key highlights
- Selected states
- Interactive affordance

Accent color is not for:
- Every heading
- Every icon
- Every surface
- Every border

If everything is accented, nothing is.

## Playful-Comic Restraint

Because Food Run has playful visual DNA:
- Accent usage should be concentrated and meaningful
- Saturation should be controlled
- Whimsy should come from the full system, not color overload
- Color should support trust, not fight it

## Semantic State Colors

State colors must feel:
- Readable
- Serious enough to trust
- Distinct from each other
- Consistent across the app

Do not use novelty color choices that weaken meaning.

## Contrast Rules

All text and interactive states must maintain clear contrast.

Especially check:
- Helper text
- Disabled text
- Placeholder-adjacent copy
- Focus rings
- Secondary buttons
- Dark-mode muted text

If a state looks polished but is hard to read, it is not finished.

## Dark-Mode Rules

Dark mode should preserve:
- Clarity
- Warmth
- Hierarchy
- Contrast
- Page rhythm

Do not:
- Invert blindly
- Let accents become neon
- Let dark surfaces collapse into one flat field
- Sacrifice readability for mood

## Theme Consistency Rules

When making color changes:
- Update semantic roles, not one-off values
- Preserve consistent meaning between pages
- Verify hover, focus, active, and disabled states
- Ensure dark and light mode remain emotionally aligned

## Review Checklist

Before approval, check:
- Does color guide attention appropriately?
- Are accents restrained?
- Do neutrals carry the layout well?
- Do state colors feel trustworthy?
- Does dark mode keep the same product personality?
- Does the page still feel calm?
- Are multiple bordered surfaces creating unnecessary visual noise through color contrast alone?
