# Review Rubric

## Purpose

This rubric is the standard framework for reviewing Food Run UI and UX.

Use it during:
- Design critique
- Implementation review
- Pre-demo review
- Pre-release review
- Post-change regression review

## Scoring Model

Each category can be judged informally or with a simple rating:
- Strong
- Acceptable
- Weak
- Failing

Use written evidence, not vague impressions.

## 1) Dominant Job Clarity

Questions:
- What is the user trying to do on this page?
- Is that obvious within a few seconds?
- Is the primary action easy to identify?

Weak signs:
- Too many equal regions
- Confusing entry point
- Unclear action language

## 2) Information Hierarchy

Questions:
- What draws the eye first?
- What draws the eye second?
- Does the scan order match the intended flow?
- Are secondary elements properly quiet?

Weak signs:
- Everything looks equally important
- Helper text competes with headings
- Secondary actions compete with primary actions

## 3) Spacing and Layout Rhythm

Questions:
- Does the page feel clean and deliberate?
- Are related items grouped clearly?
- Are section breaks strong enough?
- Are there unnecessary containers?

Weak signs:
- Cramped layout
- Empty but unstructured whitespace
- Nested surfaces
- Inconsistent alignment

## 4) Typography Quality

Questions:
- Are type roles clearly differentiated?
- Are controls readable immediately?
- Does the comic-influenced type still feel polished?
- Is helper text readable?

Weak signs:
- Blurred hierarchy
- Overly bold playful fonts
- Too many sizes
- Tiny support text

## 5) Call-to-Action Quality

Questions:
- Is there one clear primary action?
- Are button labels specific?
- Are secondary actions properly subordinate?

Weak signs:
- Many primary-feeling buttons
- Vague labels
- Action placement that interrupts the natural flow

## 6) Trust and Scope Honesty

Questions:
- Does the page sound believable?
- Does it imply capabilities that do not exist?
- Does it handle uncertainty honestly?

Weak signs:
- Fake confidence
- Fake automation sophistication
- Fake persistence or sync language

## 7) Content and Microcopy

Questions:
- Are labels direct?
- Is helper text useful?
- Do errors explain next steps?
- Do empty states guide the user forward?

Weak signs:
- Long explanatory copy
- Generic labels
- Useless empty states
- Robotic or overhyped tone

## 8) Interaction States

Questions:
- Are hover, focus, active, and disabled states clear?
- Are loading and error states calm and honest?
- Do interactions feel polished?

Weak signs:
- Hover-only discoverability
- Invisible focus
- Weak disabled differentiation
- Arbitrary motion

## 9) Responsive Behavior

Questions:
- Does the page stay clear on narrow widths?
- Does the primary task remain obvious on mobile?
- Do buttons, text, and layout still feel balanced?

Weak signs:
- Awkward wrapping
- Compressed actions
- Broken hierarchy on small screens

## 10) Accessibility Basics

Questions:
- Is focus-visible strong?
- Is text readable?
- Are labels explicit?
- Do state cues rely on more than color?

Weak signs:
- Faint muted text
- Placeholder-only fields
- Weak focus treatment
- Inaccessible touch targets

## 11) Surface Discipline

Questions:
- Are bordered surfaces actually earning their place?
- Could spacing replace one or more boxes?
- Is an empty state over-boxed?
- Does the page feel card-heavy because of prototype carryover?

Weak signs:
- Every region has its own panel
- Simple content is boxed multiple times
- Borders are doing work that spacing should do
- Old reviewer-shell card patterns remain without a user-centered reason

## 12) Food Run Brand Fit

Questions:
- Does the page still feel like Food Run?
- Does it preserve playful warmth?
- Is the personality controlled and credible?
- Is it calm enough for recurring use?

Weak signs:
- Generic SaaS drift
- Childish clutter
- Over-decorated charm
- Sterile loss of brand voice

## Final Review Questions

1. Would a first-time user understand where to start?
2. Would a repeat user feel this is efficient enough for weekly use?
3. Does this surface feel more trustworthy after the change?
4. Is the polish coming from precision rather than decoration?
5. Would this be safe to show to live users?
6. Did we keep only the surfaces that are truly helping the user?
