# Typography

## Purpose

Typography is one of Food Run’s strongest identity levers.

Because the product uses a playful-comic token system, typography must be handled with unusual discipline. The wrong usage will make the interface feel childish or noisy. The right usage will make it feel warm, distinct, and polished.

## Type Goals

The type system should feel:
- Readable
- Warm
- Stable
- Lightly expressive
- Disciplined
- Modern enough for production use
- Distinctive without becoming a novelty act

## Primary Type Direction

Use the current comic-leaning font system in a controlled way.

That means:
- Body text and controls must remain easy to read at production sizes
- Type contrast should come from role and size, not constant weight escalation
- Playful faces should not be used to justify weak hierarchy
- The comic influence should remain a brand accent, not a readability liability

## Role Model

Keep the number of text roles limited and clear.

Recommended roles:
- Display
- Page title
- Section title
- Item title
- Body
- Secondary body
- Field label
- Helper text
- Button label
- Micro or status text

## Type-Scale Guidance

Use fewer sizes with stronger distinction.

Suggested principle:
- One size for body copy
- One smaller size for helper and micro text
- One size for controls if body sizing is not sufficient
- Two to three heading tiers only

Avoid introducing many neighboring sizes that blur the hierarchy.

## Button and Control Typography

Buttons should feel:
- Readable
- Intentional
- Stable
- Confident without shouting

Guidance:
- Do not over-tighten tracking
- Do not over-bold playful fonts until counters close up
- Maintain enough size for immediate scanning
- Ensure mixed icon-and-text compositions have clean spacing
- Use sentence case unless there is a strong reason otherwise

## Heading Guidance

Headings should:
- Establish structure quickly
- Feel warm and modern
- Avoid theatrical styling
- Avoid oversized dominance unless the page truly has one simple task

Prefer strong spacing and role contrast over dramatic weight jumps.

## Line Length and Measure

Body text should avoid extremely long lines.

General direction:
- Short to medium measure for helper and instructional text
- Moderate measure for body paragraphs
- Extremely long rows should be constrained even on desktop

The goal is low-friction scanning.

## Line-Height Guidance

Use generous but controlled line height.

Too tight:
- Feels crowded
- Makes comic-leaning fonts harder to read

Too loose:
- Weakens grouping
- Reduces perceived polish

Body text should feel breathable but compact enough for workflow UI.

## Font Pairing Rules

If using multiple fonts or fallback families:
- Keep roles consistent
- Do not mix many distinct personalities in one view
- Ensure fallback behavior does not break the tone completely
- Check dark mode and smaller sizes carefully

## Emphasis Rules

Prefer emphasis in this order:
1. Layout and grouping
2. Size role
3. Weight
4. Color
5. Decoration

Avoid solving every hierarchy problem with bold text.

## Playful-Comic Safety Rules

Because Food Run is playful-comic:
- Never let the type feel sloppy
- Never let whimsy reduce task clarity
- Never overuse oversized playful headlines
- Never combine too many visually loud type treatments

The interface should feel charming, not unserious.

## Typography Checklist

Before approving a page, check:
- Are the roles obvious?
- Are controls readable immediately?
- Is body text easy to scan?
- Is helper text truly subordinate?
- Does the page title feel proportionate?
- Does the font still feel polished on mobile?
- Does the comic influence support rather than weaken trust?

## Default Typography Decisions to Preserve

Prefer:
- Stable body size
- Clear button size
- A modest number of heading levels
- Semantic naming of type tokens
- Role-based hierarchy rather than one-off overrides

Avoid:
- Tiny helper text
- Overly bold buttons
- Too many size steps
- Novelty text treatments
- Crowded title rows with poor icon spacing
