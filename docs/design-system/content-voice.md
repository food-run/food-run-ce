# Content Voice

## Purpose

Food Run’s visual polish depends heavily on writing quality.

The interface should sound:
- Direct
- Warm
- Grounded
- Useful
- Trustworthy

It should not sound:
- Vague
- Robotic
- Overhyped
- Fake-smart
- Overly jokey
- Overly corporate

## Writing Goals

UI writing should:
- Reduce hesitation
- Make actions obvious
- Explain uncertainty honestly
- Help users recover quickly
- Support the product’s calm and friendly personality

## Core Voice Traits

### Direct
Say what the action or state actually is.

### Warm
Use human language without becoming chatty or performative.

### Specific
Name the outcome, not a generic category of action.

### Honest
Do not overstate system certainty or sophistication.

### Lightweight
Do not make users read long blocks to perform simple actions.

## Button-Label Rules

Prefer button labels that describe the result.

Good direction:
- Import recipe
- Add to planner
- Update list
- Review ingredients

Weak direction:
- Submit
- Continue
- Process
- Confirm

## Helper-Text Rules

Helper text should:
- Exist only when needed
- Remove uncertainty
- Stay short
- Sit close to the relevant control

Helper text is not a dumping ground for product explanation.

## Error-Text Rules

Good error text includes:
1. What went wrong
2. What the user can do next

Avoid:
- Blame-heavy language
- Vague technical phrasing
- Unexplained failure states

## Empty-State Rules

An empty state should:
- Explain why the area is empty
- Point toward the next useful action
- Preserve momentum

Avoid empty states that are only decorative or apologetic.

If an empty state does not need a full boxed treatment, keep it lighter.

## Status-Text Rules

Status language should be calm and believable.

Do not:
- Use inflated success language
- Imply background sophistication that does not exist
- Present uncertain outputs as definitive

## Scope-Honesty Rules

When the product is incomplete or static:
- Say so clearly when needed
- Avoid visual or verbal overclaiming
- Prefer grounded descriptions over product-marketing theatre

## Playful-Comic Voice Rules

Food Run can be playful, but the voice should remain credible.

Allowed:
- Light warmth
- Approachable phrasing
- Friendly guidance

Avoid:
- Cutesy overload
- Meme references
- Excessive exclamation
- Novelty that weakens trust

## Writing Review Checklist

Before approval, check:
- Is the call to action specific?
- Is any helper text actually useful?
- Does any copy delay action unnecessarily?
- Do errors explain what to do next?
- Does the writing sound honest?
- Does the voice still match Food Run?
- Is any boxed message acting as a substitute for clearer writing?
